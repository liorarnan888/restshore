# Product Backlog

## Calendar Creation Experience

- Add more perceived-progress motion between real sync jumps so the build feels continuous.
- Add richer intermediate phases and micro-feedback during Google Calendar creation.
- Keep refining the "wow moment" copy so it feels personal, calm, and premium.

## Calendar Personalization

- Personalize which events appear based on intake answers.
- Remove unnecessary events for users who do not need them, such as caffeine cutoffs for non-caffeine users.
- Adapt event descriptions to reflect the user's specific reported patterns and likely contributors.
- Add stronger personalization for bedtime, overnight awakenings, racing thoughts, and behavior triggers.

## Report, Email, and Education

- Add a clearer explanation of what CBT-I is, how effective it is, how to use the calendar, and what the user needs to do for success.
- Add links to supporting materials and related resources.
- Add a future main-page summary area for user updates and plan changes.
- Add a clinician-facing escalation summary page that the user can choose to share with a doctor, with sleep history, key flags, recent diary patterns, current plan, and reasons for escalation.

## Competitor and Marketing Research

- Run competitor research on sleep apps, CBT-I products, and adjacent wellness flows to understand positioning, funnels, trust cues, and feature framing.
- Run UI inspiration research specifically for marketing pages so the public-facing experience feels premium, differentiated, and conversion-oriented.
- Translate the competitor and UI research into a future marketing backlog covering landing page structure, proof elements, testimonials/social proof placeholders, and doctor-trust messaging.

## Reminder Strategy

- Revisit reminder strategy across calendar events, check-ins, and emails to avoid overload.
- Decide which reminders should be real calendar events versus softer prompts or email nudges.

## Daily Updates / Sleep Diary

- Add a very short mobile-first morning sleep check-in.
- Use the check-in as a sleep diary layer before doing heavier plan adaptation.
- Preserve planned sleep events while reflecting logged outcomes in a visible but non-destructive way.
- Rename past sleep events to `Sleep` after logging and surface the logged outcome in both the title and description without rewriting the planned history.
- Add future-plan updates only to upcoming events, never to past events.
- Add a future main-page summary of changes made to the user's plan.
- Delay in-app update cards until a richer main page exists.

## Daily Updates Product Direction

- Phase 1: morning check-in, sleep diary storage, easy calendar entry point, and visible logged outcome for the previous night.
- Phase 2: micro-adjustments to upcoming coaching text and bedtime/wind-down guidance based on recent nights.
- Phase 3: structural weekly adjustments after enough diary data, with plan versioning and change summaries.
